# Contributing to Alina Vista Boot

Thank you for your interest in contributing to **Alina Vista Boot**.

This project is source-available and community-driven, but it is **not open-source**
in the OSI sense. Please read the following guidelines carefully before contributing.

---

## Scope of the project

Alina Vista Boot is a **boot firmware** designed to:

- manage device startup
- handle WiFi provisioning
- perform secure OTA updates of the application firmware

It is a critical system component.  
Contributions should respect the existing architecture and design goals.

---

## What kind of contributions are welcome

You are welcome to contribute:

- bug fixes
- code refactoring
- documentation improvements
- performance or memory optimizations
- security improvements
- optional or experimental features (clearly marked)

Please keep changes focused and minimal.

---

## Contribution rules

- All contributions must target the `main` branch unless otherwise specified.
- Code must compile using **PlatformIO**.
- Avoid introducing new dependencies unless strictly necessary.
- Follow the existing code style and structure.
- Keep the boot firmware lightweight (flash and RAM usage matter).

---

## Pull Request process

1. Fork the repository
2. Create a feature or fix branch
3. Commit your changes with clear commit messages
4. Open a Pull Request describing:
   - what the change does
   - why it is needed
   - any potential impact on boot behavior

The maintainer may request changes before merging.
